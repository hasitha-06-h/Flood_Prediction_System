Machine Learning Workflow

From Raw Weather Data to a Flood Verdict

Every prediction on this site passes through the same five-stage pipeline that was used to train the deployed model, so what you see on the Predict page is the exact process the dissertation documents.

Data Collection

115 historical readings covering annual and seasonal rainfall, temperature, humidity and cloud cover were compiled from meteorological records.

Cleaning & Preprocessing

Missing values are median-imputed, outliers are capped using the IQR method, and every feature is standardised with a fitted scaler.

Model Training

Decision Tree, Random Forest, KNN and XGBoost are each trained on an 80/20 split of the cleaned dataset.

Benchmarking

Accuracy, precision, recall and F1 score are compared across all four models, and the strongest performer is selected automatically.

Deployment

The winning model and its scaler are serialised with Joblib and served through this Flask application in real time.

Model Comparison

A transparent look at how each algorithm performed on the held-out test split before one was chosen for deployment.

Project Features

Built for Real Early-Warning Scenarios

Early Warning

A meteorologist enters live readings from a flood-prone district and receives an immediate risk classification, giving authorities lead time to act before water levels rise.

Resource Allocation

Disaster management teams can run readings for several regions back to back and prioritise rescue and relief resources toward the highest-risk locations first.

Validated Accuracy

Every prediction is backed by a model that was benchmarked against three other algorithms on historical flood records before being chosen for deployment.
